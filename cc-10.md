Program 10: Demonstrate EC2 Auto Scaling Group using Default VPC

Objective

Create an Application Load Balancer (ALB) and an Auto Scaling Group (ASG) that automatically launches and maintains EC2 instances.

---

Step 1: Create a Security Group for EC2

Navigate to:

EC2 → Security Groups → Create Security Group

Configuration

Setting| Value
Name| "asg-web-sg"
VPC| Default VPC

Inbound Rules

Type| Port| Source
SSH| 22| My IP
HTTP| 80| "0.0.0.0/0"

Click Create Security Group.

---

Step 2: Create a Target Group

Navigate to:

EC2 → Target Groups → Create Target Group

Configuration

- Target Type: Instances
- Protocol: HTTP
- Port: 80
- VPC: Default VPC
- Target Group Name: "web-target-group"

Click Next.

⚠️ Do not register any targets manually.

Click Create Target Group.

---

Step 3: Create an Application Load Balancer

Navigate to:

EC2 → Load Balancers → Create Load Balancer

Select:

- Application Load Balancer

Basic Configuration

Setting| Value
Name| "web-alb"
Scheme| Internet-facing
IP Address Type| IPv4
VPC| Default VPC

Availability Zones

Select at least two default public subnets in different Availability Zones.

Security Group

Create or select a security group with:

Type| Port| Source
HTTP| 80| "0.0.0.0/0"

Listener

- Protocol: HTTP
- Port: 80

Default Action

Forward traffic to:

web-target-group

Click Create Load Balancer.

---

Step 4: Create a Launch Template

Navigate to:

EC2 → Launch Templates → Create Launch Template

Configuration

Setting| Value
Name| "web-template"
AMI| Ubuntu 24.04 LTS
Instance Type| "t2.micro"
Key Pair| Select your existing key pair

Network Settings

- Auto-assign Public IP: Enable
- Security Group: "asg-web-sg"

Advanced Details → User Data

Paste the following script:

#!/bin/bash
sudo apt update -y
sudo apt install apache2 -y

echo "<h1>Auto Scaling Demo</h1>
<p><b>Hostname:</b> $(hostname)</p>
<p><b>Private IP:</b> $(hostname -I | awk '{print $1}')</p>" | sudo tee /var/www/html/index.html > /dev/null

sudo systemctl enable apache2
sudo systemctl restart apache2

Click Create Launch Template.

---

Step 5: Create an Auto Scaling Group

Navigate to:

EC2 → Auto Scaling Groups → Create Auto Scaling Group

Basic Details

Setting| Value
Name| "web-asg"
Launch Template| "web-template"

Click Next.

---

Network

Select:

- VPC: Default VPC
- Subnets: Select 2 default public subnets

Click Next.

---

Attach Load Balancer

Choose:

- Attach to an existing load balancer

Select:

- Load Balancer: "web-alb"
- Target Group: "web-target-group"

Enable:

- ✅ Turn on Elastic Load Balancing health checks

Click Next.

---

Configure Group Size

Setting| Value
Desired Capacity| 2
Minimum Capacity| 2
Maximum Capacity| 3

Click Next until Create Auto Scaling Group, then click Create.

---

Step 6: Verify EC2 Instances

Wait 2–5 minutes and navigate to:

EC2 → Instances

You should see 2 running EC2 instances launched automatically by the Auto Scaling Group.

---

Step 7: Test the Load Balancer

Navigate to:

EC2 → Load Balancers

Copy the DNS Name of the ALB.

Open it in a browser:

http://<ALB-DNS-NAME>

Expected output:

Auto Scaling Demo

Hostname: ip-172-31-xx-xx
Private IP: 172.31.xx.xx

Refresh the page multiple times to observe requests being served by different EC2 instances.

---

Step 8: Test Auto Scaling

1. Go to:
   
   EC2 → Instances

2. Select one instance created by the Auto Scaling Group.

3. Click:
   
   Instance State → Terminate

4. Wait 2–5 minutes.

The Auto Scaling Group will automatically launch a replacement instance to maintain the desired capacity of 2 instances.

---

Result

- ✅ Created an Application Load Balancer
- ✅ Created a Launch Template
- ✅ Created an Auto Scaling Group
- ✅ Automatically launched 2 EC2 instances
- ✅ Successfully served the application through the Load Balancer
- ✅ Verified that terminating an instance causes the Auto Scaling Group to automatically launch a replacement instance
