Program 10: Demonstrate EC2 Auto Scaling Group using Default VPC

Objective

Create an Application Load Balancer (ALB) and an Auto Scaling Group (ASG) that automatically maintains EC2 instances.

---

Step 1: Create a Security Group for EC2

Go to:

EC2 → Security Groups → Create Security Group

Configure:

Setting| Value
Name| "asg-web-sg"
VPC| Default VPC

Inbound Rules

Type| Port| Source
SSH| 22| My IP
HTTP| 80| Anywhere (0.0.0.0/0)

Create the security group.

---

Step 2: Create a Target Group

Go to:

EC2 → Target Groups → Create Target Group

Select:

- Target type: Instances
- Protocol: HTTP
- Port: 80
- VPC: Default VPC

Target Group Name:

web-target-group

Click Next.

Do NOT register any targets now.

Click Create Target Group.

---

Step 3: Create a Load Balancer

Go to:

EC2 → Load Balancers → Create Load Balancer

Choose:

Application Load Balancer

Settings:

Field| Value
Name| "web-alb"
Scheme| Internet-facing
IP Type| IPv4
VPC| Default VPC

Select Subnets

Choose at least two default public subnets in different Availability Zones.

Security Group

Create or select one with:

Type| Port| Source
HTTP| 80| Anywhere (0.0.0.0/0)

Listener

- Protocol: HTTP
- Port: 80

Default Action

Forward to:

web-target-group

Click Create Load Balancer.

---

Step 4: Create Launch Template

Go to:

EC2 → Launch Templates → Create Launch Template

Settings:

Field| Value
Name| "web-template"
AMI| Ubuntu 24.04
Instance Type| t2.micro
Key Pair| Select your key pair

Network Settings

- Auto-assign Public IP: Enable
- Security Group: "asg-web-sg"

Advanced Details → User Data

Paste:

#!/bin/bash
sudo apt update -y
sudo apt install apache2 -y

echo "<h1>Auto Scaling Demo</h1>
<p><b>Hostname:</b> $(hostname)</p>
<p><b>Private IP:</b> $(hostname -I | awk '{print $1}')</p>" | sudo tee /var/www/html/index.html > /dev/null

sudo systemctl enable apache2
sudo systemctl restart apache2

Click Create Launch Template.

---

Step 5: Create Auto Scaling Group

Go to:

EC2 → Auto Scaling Groups → Create Auto Scaling Group

Basic Details

Field| Value
Name| "web-asg"
Launch Template| "web-template"

Click Next.

---

Network

Select:

- VPC → Default VPC
- Availability Zones → Select 2 default subnets

Click Next.

---

Attach Load Balancer

Choose:

Attach to an existing load balancer

Select:

web-alb

Choose Target Group:

web-target-group

Enable:

Turn on Elastic Load Balancing health checks

Click Next.

---

Group Size

Set:

Setting| Value
Desired Capacity| 2
Minimum Capacity| 2
Maximum Capacity| 3

Click Next until Create Auto Scaling Group.

---

Step 6: Wait for Instances

After 2–5 minutes:

EC2 → Instances

You should see 2 running instances launched by the ASG.

---

Step 7: Test the Load Balancer

Go to:

EC2 → Load Balancers

Copy the DNS Name.

Open in browser:

http://<ALB-DNS-NAME>

You should see a page similar to:

Auto Scaling Demo

Hostname: ip-172-31-xx-xx
Private IP: 172.31.xx.xx

Refresh several times to observe requests being served by different instances.

---

Step 8: Test Auto Scaling Recovery

1. Go to:

EC2 → Instances

2. Select one instance created by the ASG.

3. Click:

Instance State → Terminate

4. Wait 2–5 minutes.

The Auto Scaling Group will automatically launch a replacement instance to maintain the desired capacity of 2.

---

Result

- ✅ Application Load Balancer created
- ✅ Launch Template created
- ✅ Auto Scaling Group created
- ✅ 2 EC2 instances launched automatically
- ✅ Web application accessible through ALB
- ✅ When one instance is terminated, Auto Scaling automatically creates a new instance
